# Content Strategy

## 1. Purpose

This document defines the content governance model for the HSC-AI platform. It covers how content enters the system, how it is owned and classified, how it moves through its lifecycle, and the quality standards that must be met before publication.

All content on this platform — questions, writing prompts, explanations — directly affects the learning outcomes of children preparing for high-stakes NSW exams. Quality and legal compliance are non-negotiable.

---

## 2. Content Sources

### 2.1 Manual Authoring

Content written directly by a platform administrator.

- Default ownership classification: `original`
- Enters lifecycle at: `draft`
- Review required: yes, by a second admin (or same admin in V1 single-admin setup, with all actions logged)

Manual authoring is the most reliable source of clean, compliant content. Quality depends on the author's subject knowledge and adherence to the quality framework in section 5.

### 2.2 OCR Import

Content extracted from PDF, image, or scanned document sources.

- Default ownership classification: `internal_draft`
- Enters lifecycle at: `draft` (after OCR and AI structure extraction)
- Review required: yes — mandatory. Admin must confirm copyright status before assigning a publishable classification.

OCR produces raw extracted text and an AI-structured draft. Neither is automatically correct. The reviewing admin must:

1. Compare the original source page against the extracted draft.
2. Verify the correct answer is identified accurately.
3. Confirm or assign the copyright status.
4. Edit any extraction errors.

OCR-imported content from official NSW DoE exam papers cannot be published unless a valid licence is held. See Section 4 for classification rules.

### 2.3 AI Generated

Content generated by an AI provider under admin direction.

- Default ownership classification: `original` (platform owns content generated through its own prompts)
- Enters lifecycle at: `draft`
- Review required: yes — mandatory. No AI-generated content may be auto-published.

AI generation is a productivity tool for seeding the question bank. It does not replace editorial review. AI-generated questions frequently have:

- Subtly incorrect answers (especially maths)
- Ambiguous distractors
- Inappropriate difficulty calibration
- Curriculum misalignment

Every AI-generated question must be reviewed against the quality framework before approval.

## 3. Content Ownership Types

Every piece of content must carry exactly one ownership classification. This classification determines publishing eligibility and informs copyright obligations.

| Classification | Description | Who Assigns | Publishing Allowed |
|---|---|---|---|
| `original` | Written by a platform admin or AI under platform direction | Admin at creation | Yes |
| `licensed` | Third-party content with a confirmed written licence | Admin with licence reference | Yes |
| `public_domain` | Confirmed as public domain (e.g., very old published materials, expired copyright) | Admin with documentation | Yes |
| `approved_internal` | Internal draft that completed the full review process and was cleared | Admin (final review step) | Yes |
| `user_provided_with_rights` | Submitted by an external party with an explicit declaration of rights | Admin after reviewing declaration | Yes, with caution |
| `internal_draft` | Copyright status not confirmed; pending review | System default for OCR import | No |
| `restricted_reference_only` | Known copyright-protected content; held for admin reference only | Admin | No |

### Publishing Restriction

The system enforces a hard block on publishing questions classified as `internal_draft` or `restricted_reference_only`. This is enforced at the API layer, not just the UI.

An admin cannot override this block by reviewing and approving a question without first changing the ownership classification to a publishable type.

### NSW Government Exam Content

Official NSW OC and Selective past exam papers are published by the NSW Department of Education. They are copyright-protected.

- Scanning and storing for internal admin reference: permissible if access is restricted to admins.
- Publishing reproduced questions to students: requires a written licence from NSW DoE.
- Writing original questions inspired by exam style: permissible.
- Prompting AI with specific questions from official papers: not recommended; grey area under copyright law.

If in doubt, classify as `restricted_reference_only` and seek legal advice.

## 4. Content Lifecycle

All content — regardless of source — follows the same lifecycle.

```
┌─────────────────────────────────────────────────────────────┐
│                     CONTENT LIFECYCLE                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [Manual / OCR / AI]                                       │
│          ↓                                                  │
│       DRAFT                                                 │
│   Content created; not visible to students.                 │
│   Admin may edit freely.                                    │
│          ↓                                                  │
│       REVIEW                                                │
│   Admin has submitted for review.                           │
│   Ownership classification required.                        │
│   Quality checks applied.                                   │
│   A second admin should review (V2+; single admin in V1).   │
│          ↓                    ↑                             │
│       APPROVED          REJECTED (with note)                │
│   Passed review.              Content returned to admin     │
│   Ownership confirmed.        with rejection reason.        │
│   Ready to add to exam sets.  Admin may edit and resubmit.  │
│          ↓                                                  │
│       PUBLISHED                                             │
│   Assigned to at least one published ExamInstance.          │
│   Visible to entitled students.                             │
│   Content is now locked (version-controlled on edit).       │
│          ↓                                                  │
│       ARCHIVED                                              │
│   Removed from active exam sets.                            │
│   No longer shown to new students.                          │
│   Historical attempt references preserved.                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### State Transition Rules

| From | To | Allowed | Condition |
|---|---|---|---|
| `draft` | `review` | Yes | Ownership classification assigned |
| `review` | `approved` | Yes | Admin approves; ownership must be publishable |
| `review` | `rejected` | Yes | Admin rejects with note |
| `rejected` | `draft` | Yes | Admin decides to revise |
| `approved` | `published` | Yes | Added to a published ExamInstance |
| `approved` | `archived` | Yes | Admin decision |
| `published` | `archived` | Yes | Admin removes from all active exam sets |
| `published` | `draft` (via new version) | Yes | Admin edits create new version; original stays published |

### Immutability After Publishing

Once a `QuestionVersion` has been delivered in a published `ExamInstance`, it is immutable. If an admin must correct a published question:

1. A new `QuestionVersion` is created with the correction.
2. The new version goes through review and approval.
3. Future exam instances reference the new version.
4. Past exam instances and all associated attempt answers continue to reference the original version — this is intentional and correct.

## 5. Quality Framework

### 5.1 Mandatory Fields

Every question approved for publishing must have all of the following:

| Field | Requirement |
|---|---|
| `stem` | Non-empty, grammatically correct, unambiguous |
| `correct_answer` | Clearly identified; must be unambiguously correct |
| `full_explanation` | Must explain why the answer is correct AND why common wrong answers are incorrect |
| `subject_id` | Assigned to a valid subject |
| `topic_id` | Assigned to a specific topic |
| `exam_type_id` | OC or Selective (V1) |
| `year_level` | 4, 5, or 6 |
| `difficulty` | easy, medium, or hard |
| `content_ownership` | A publishable classification |

### 5.2 Difficulty Calibration

Difficulty is set by the reviewing admin at approval time. Initial guidance:

| Difficulty | Description |
|---|---|
| `easy` | Most students at the target year level should answer correctly (>70% expected accuracy) |
| `medium` | Approximately half of students at the target year level should answer correctly (40–70% expected accuracy) |
| `hard` | Only high-performing students at the target year level should answer correctly (<40% expected accuracy) |

Difficulty calibration is subjective at creation time. Over time, actual student performance data (from analytics) should be used to recalibrate. The `TopicPerformance` and `QuestionAttempt` analytics entities support this recalibration in future.

### 5.3 Topic Tagging Requirements

Every question must be tagged to a topic. Topics are hierarchical:

```
Subject
  └── Topic
       └── Sub-topic (optional)
            └── SkillTag (optional)
```

Example:

```
Mathematics (Subject)
  └── Number (Topic)
       └── Fractions (Sub-topic)
            └── Adding unlike fractions (SkillTag)
```

Tagging at the `SkillTag` level is ideal but not mandatory for V1. Minimum requirement: `Subject` and `Topic`.

### 5.4 Curriculum Mapping

Questions should align with the NSW Education Standards Authority (NESA) curriculum outcomes for the relevant year level.

V1 does not implement formal NESA outcome mapping. Admins should be aware of year-level curriculum expectations when authoring or reviewing questions.

Curriculum mapping fields may be added in V2 when there is sufficient content volume to make structured alignment valuable.

### 5.5 Explanation Quality Requirements

Full explanations are mandatory. Quality criteria:

- Explains the correct answer in plain language accessible to the target year level.
- Addresses the most common incorrect choices (why they are wrong).
- Does not simply restate the question.
- Avoids jargon that a Year 4–6 student would not understand.
- For maths questions: shows the working or method, not just the answer.

Explanations are the primary learning tool for students reviewing their performance. A question with a poor explanation has lower educational value even if the question itself is good.

### 5.6 MCQ Distractor Quality

For multiple choice questions, distractors (wrong answers) must be:

- Plausible: a student who misunderstands the concept might choose them.
- Not obviously wrong: distractors that no reasonable student would choose add no value.
- Not ambiguously correct: a distractor that could be argued as a second correct answer must be revised.

Common distractor patterns for maths:

- Common calculation error (e.g., forgetting to carry a digit)
- Partial answer (e.g., finding one part of a two-step problem)
- Inverted operation (e.g., subtracting instead of adding)

### 5.7 Age-Appropriateness Review

Content must be appropriate for the target age group (9–12 years):

- No violence, adult themes, or culturally insensitive content.
- Language complexity appropriate for the year level.
- Contexts and scenarios familiar or accessible to NSW primary school children.
- Avoid contexts that systematically disadvantage particular cultural or socioeconomic groups.

## 6. Admin Responsibilities

In V1, all admin accounts share the same permission level. However, the following responsibility model should be applied operationally and will be enforced by role separation in a future release.

### 6.1 Author

Responsible for:

- Creating question drafts (manual entry or triggering AI generation)
- Setting initial topic, difficulty, and exam type metadata
- Uploading OCR source files
- Reviewing AI-extracted drafts for OCR imports
- Assigning initial ownership classification

An author must not approve their own content. In V1 with a single admin, this constraint is relaxed, but all author and reviewer actions are logged against the admin's account for audit purposes.

### 6.2 Reviewer

Responsible for:

- Checking content against the quality framework (Section 5)
- Confirming the ownership classification is accurate and appropriate
- Approving or rejecting questions with a written reason
- Flagging questions with potential copyright issues

The reviewer is the quality gate. An approval without reading the full question, explanation, and source attribution is a process failure.

### 6.3 Publisher

Responsible for:

- Assembling approved questions into ExamTemplates and ExamInstances
- Setting section structure, timing, and ordering
- Reviewing the overall exam set for balance (difficulty distribution, topic coverage)
- Publishing ExamInstances to make them available to students

In V1, the Publisher role is performed by any admin. Future: Publisher should be a separate role from Author and Reviewer to maintain editorial independence.

## 7. Content Volume Targets

The following are recommended minimum content volumes before public launch:

| Exam Type | Subject | Minimum Questions |
|---|---|---|
| OC | Mathematical Reasoning | 80 |
| OC | Thinking Skills | 60 |
| OC | Reading | 60 |
| Selective | Mathematical Reasoning | 100 |
| Selective | Thinking Skills | 80 |
| Selective | Reading | 80 |
| Selective | Writing | 10 prompts |

These targets support at least 4 distinct practice exam sets per exam type, with minimal question repetition. Lower volumes are acceptable for a closed beta launch with a small initial user cohort.

## 8. Content Maintenance

### 8.1 Annual Review

All published questions should be reviewed annually against:

- Current NESA curriculum for the relevant year level
- Any changes to the OC or Selective test format announced by NSW DoE
- Student performance data (questions with anomalous difficulty or answer distribution)

### 8.2 Question Retirement

A question should be archived when:

- It contains an error that cannot be corrected without disrupting in-flight exam instances
- It has been superseded by a better version
- The topic or format is no longer relevant to the target exam
- Student performance data suggests the question is ambiguous (e.g., unexpectedly high answer distribution across all options)

### 8.3 Copyright Audits

All questions with `licensed` or `user_provided_with_rights` classifications should be audited periodically to confirm:

- The licence is still valid
- The licence terms still permit the current use
- Attribution requirements are being met
